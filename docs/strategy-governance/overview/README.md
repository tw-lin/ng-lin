## 🎯 專案簡介

### GigHub - 工地施工進度追蹤管理系統

GigHub 是一個現代化、企業級的工地施工進度追蹤管理系統，專為營造業與工程管理團隊設計。系統採用多租戶架構，支援組織、團隊與夥伴協作，提供完整的任務管理、進度追蹤與資料分析功能。

**核心特色**:
- 🔐 **多租戶架構**: 支援多組織與多藍圖隔離
- 🚀 **即時協作**: 基於 Firebase 的即時資料同步
- 📱 **跨平台支援**: Web、行動裝置響應式設計
- 🔒 **企業級安全**: Firestore Security Rules 資料隔離
- 📊 **智能分析**: 進度追蹤與數據視覺化
- 🎨 **玄武主題**: 專業、穩定、高級的設計語言

## 🎯 業務目標

### 主要目標

1. **提升施工效率**
   - 即時任務指派與追蹤
   - 減少溝通成本
   - 優化資源配置

2. **強化協作能力**
   - 多方協作 (業主、承包商、夥伴)

4. **降低管理成本**

## 🔒 關鍵限制

### 技術限制

1. **Firebase 配額**
   - Firestore 讀寫配額
   - Cloud Functions 執行時間
   - Storage 容量限制
   - 參考：[成本優化](../deployment(部署)/05-cost-optimization.md)

2. **瀏覽器支援**
   - Chrome 90+
   - Firefox 88+
   - Safari 14+
   - Edge 90+
   - 不支援 IE11

3. **檔案大小限制**
   - 單一檔案: 最大 50MB
   - 總附件大小: 每任務 200MB

### 業務限制

1. **使用者限制**
   - 免費版: 最多 10 個使用者
   - 專業版: 最多 100 個使用者
   - 企業版: 無限制

2. **藍圖限制**
   - 免費版: 最多 3 個藍圖
   - 專業版: 最多 50 個藍圖
   - 企業版: 無限制

3. **資料保留**
   - 審計日誌: 保留 1 年
   - 已刪除項目: 保留 30 天
   - 備份: 保留 90 天

### 安全限制

1. **認證方式**
   - 僅支援 Email/Password
   - 未來支援: Google SSO, SAML

2. **資料隔離**
   - 多租戶完全隔離
   - 跨藍圖存取需明確授權
   - 參考：[Security Rules](../security(安全)/README.md)
